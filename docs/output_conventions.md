# Output Conventions

## Directory structure

```
output/
├── dashboard.html                  # Interactive HTML dashboard (src/dashboard.py::write_dashboard)
├── data/
│   ├── refinery_results.json       # Refinery pipeline results
│   ├── manuscript_variables.json   # {{TOKEN}} → value mapping
│   └── publication_ledger.json     # Publication/DOI ledger
├── reports/
│   ├── token_plan.json             # Token plan with provenance
│   ├── claim_support_registry.json # Project-local contribution-claim assay
│   ├── evidence_registry.json      # Shared template evidence validator
│   ├── figure_quality_report.json  # Per-figure quality checks
│   ├── cover_visualization.json    # Cover visualization manifest
│   ├── artifact_manifest.json      # Full artifact manifest
│   ├── output_statistics.json/.txt # Output statistics summary
│   ├── test_results.json/.md       # Test run results
│   └── validation_report.json/.md  # Output validation report
├── figures/
│   ├── purity_progression.png/.svg      # Purity progression chart
│   ├── karat_grading.png/.svg           # Karat grading scale
│   ├── token_density.png/.svg           # Token distribution
│   ├── provenance_sankey.png/.svg       # Provenance trace flow
│   ├── purity_claim_scatter.png/.svg    # Stage purity vs one project-level claim-support assay
│   ├── token_heatmap.png/.svg           # Token selection heatmap
│   ├── integrity_gate_matrix.png/.svg   # Integrity gate matrix
│   ├── formalism_traceability.png/.svg  # Formalism traceability graph
│   ├── implementation_circuit.png/.svg  # Implementation circuit graph
│   ├── claim_evidence_assay.png/.svg    # Claim-evidence assay
│   ├── integrity_risk_matrix.png/.svg   # Integrity risk matrix
│   ├── evidence_tier_ladder.png/.svg    # Evidence-tier ladder
│   ├── cover_visualization.png/.svg     # Cover visualization composite
│   └── figure_registry.json             # Figure registry (all 12 labels)
├── manuscript/                     # Hydrated manuscript (resolved tokens)
│   ├── 00_abstract.md
│   ├── 01_introduction.md
│   └── ...
├── pdf/
│   └── template_gold_refinement_combined.pdf
└── web/
    └── index.html
```

All 12 stable figure labels (see `docs/syntax_guide.md`) are generated by
`src/figures/registry.py::generate_all_figures()`, each with a PNG and SVG
companion; `output/dashboard.html` is a separate artifact owned by
`src/dashboard.py`, not part of the figures registry.

## File conventions

- **JSON files**: `indent=2`, `sort_keys=True`, `ensure_ascii=False`
- **Figures**: 300 DPI PNG, tight bounding box
- **Manuscript**: Markdown with `{{TOKEN}}` placeholders resolved
- **Registry**: `figure_registry.json` with label, path, caption per figure
- **Measurement integrity**: captions and encodings must preserve the unit and scope of each source value; do not manufacture stagewise series from project-level aggregates

## Regeneration

All output is disposable. Regenerate from source:

```bash
uv run python scripts/refinement_analysis.py
uv run python scripts/z_generate_manuscript_variables.py
```

Never hand-edit files under `output/`.
