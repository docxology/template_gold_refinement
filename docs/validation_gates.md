# Validation Gates and Statistical Integrity

## Stage 04 gate policy

Stage 04 produces a validation report and an exit status. The exit status is
fail-closed for checks that can certify artifact, provenance, or claim
integrity:

| Check | Role | Failure behavior |
| --- | --- | --- |
| PDF validation | Render integrity | Blocking |
| Transmission bookends | Manuscript page-span integrity | Blocking |
| Markdown validation | Source encoding and syntax | Blocking |
| Output structure | Required artifacts and directories | Blocking |
| Figure registry | References, generated files, and SVG companions | Blocking |
| Evidence registry | Manuscript numbers, citations, and source paths | Blocking |
| Project design overlays | Declared profile and experiment-plan contract | Blocking |
| Artifact manifest | Hash, size, and declared-output provenance | Blocking |
| Claim verification | Contradiction/insufficiency report | Blocking when enabled |
| Prose quality | AI-writing fingerprints | Advisory/report-only |

An enabled claim-verification run is green only when no claim is classified as
`contradicted` or `insufficient`; an explicitly skipped optional verifier is
recorded as skipped. A prose-quality flag is retained in the report but does
not certify or reject scientific content.

## Seed-sensitivity report contract

`output/data/seed_sensitivity.json` is generated by
`src/seed_sensitivity.py::run_seed_sensitivity`. It must contain, and the
hydration script validates by deterministic recomputation:

- the declared sample size and seed range;
- the sampling scheme and the conditional exchangeability assumption;
- the agreement interval method, threshold interval method, and precision method;
- the Hoeffding minimum sample size for the declared confidence and target radius;
- the deterministic bootstrap seed and replicate count;
- the bootstrap mean interval and the technical-replicate summary metrics;
- full SHA-256 token-plan digests, rather than truncated identifiers.

This prevents an output JSON from becoming self-authenticating evidence. The
study uses deterministic contiguous integer seeds as a declared sensitivity
surface. It supports reproducibility and bounded conditional summaries; it does
not support population-level inference about people, documents, or empirical
field data.

## Negative controls

The repository tests the failure boundaries directly:

- mutating `mean_agreement` makes the seed-report validator fail;
- a missing unreferenced generated figure fails figure validation;
- a contradicted claim makes Stage 04 return nonzero;
- undecodable manuscript bytes make Markdown/evidence validation fail closed;
- flagged prose remains advisory and does not independently fail the pipeline.

Regenerate and validate from the repository root:

```bash
SOURCE_DATE_EPOCH=1782345600 uv run python scripts/pipeline/stage_02_analysis.py \
  --project templates/template_gold_refinement
uv run python scripts/pipeline/stage_03_render.py \
  --project templates/template_gold_refinement
uv run python scripts/pipeline/stage_04_validate.py \
  --project templates/template_gold_refinement
uv run python -m infrastructure.validation.cli prerender \
  projects/templates/template_gold_refinement/manuscript --repo-root .
uv run python -m infrastructure.validation.cli evidence \
  projects/templates/template_gold_refinement \
  --manuscript-dir projects/templates/template_gold_refinement/manuscript \
  --fail-on-issues
```
