# Agent Instructions

## AI agent guidance for editing this exemplar

### Core principle

The analogy is load-bearing: each metallurgical stage maps to a real
template-infrastructure operation. Do not treat stages as decorative labels.

### Editing rules

1. **Lexicon changes**: edit `manuscript/config.yaml` under
   `gold_refinement.lexicon`, not source code.
2. **Refinery stage changes**: edit `src/refinery.py` and ensure purity is
   strictly increasing (`assert_monotone_increase`).
3. **New manuscript tokens**: add the slot in config, the variable generation
   in `src/manuscript_variables.py`, and a test in
   `tests/test_manuscript_variables.py`.
4. **Never hand-edit** files under `output/` — regenerate from source.
5. **Run the full gate** after edits:
   ```bash
   uv run pytest tests/ --cov=src --cov-fail-under=90
   uv run python scripts/refinement_analysis.py
   uv run python scripts/z_generate_manuscript_variables.py
   ```

### Forbidden actions

- Do not import `infrastructure.*` in `src/` (enforced by drift checker)
- Do not use mocks in tests (zero-mock policy)
- Do not hard-code purity values in manuscript prose (use `{{TOKEN}}`)
- Do not edit `output/manuscript/` directly (it is regenerated)

### Token protocol

Every numeric value in manuscript prose must be a `{{UPPERCASE_TOKEN}}`
generated by `src/manuscript_variables.py::generate_variables()`. The live
cross-reference test (`test_all_manuscript_tokens_are_generated`) fails CI if
any token drifts.
